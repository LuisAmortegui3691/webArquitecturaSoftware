Patrón cliente - servidor

Podemos visualizar fácilmente del modelo cliente - servidor cuando el usuario habré una página desde el navegador, en las cuales actúan una serie de procesos internos que están codificados con diferentes lenguajes de programación, etiquetas HTML y estilos CSS, los cuales una persona no puede ver per si se están compilando y ejecutando, a continuación, veremos se explicara con mayor detalle el frontend y backend, los lenguajes que existen para estos propósitos y las diferentes características.

BASES DE DATOS

En cuanto a las bases de datos tenemos tanto las bases relacionales y no relacionales, entre los diferentes BD podemos encontrar SQL Lite, Cassandra, MySQL, mongo DB, ORACLE. En cuanto a las bases de datos relacionales tienen un mayor soporte ya que llevan mucho más tiempo el mercado, están formadas por tablas que contienen campos estructurados, entre más compleja sea más procesamiento necesitara. En cuanto a las bases no relacionales, la consulta para una gran cantidad de datos se optimiza, el almacenamiento no es estructurado permitiendo gran escalabilidad, no se necesitan tantos recursos para ejecutarse a diferencia de las relacionales que si las necesitan.

BACKEND

El backend es el lenguaje de todo desarrollador de software, es lo que el usuario no puede ver y en la mayoria de los casos no lo pueden entender. En la practica se le conoce como el codigo de programacion que corre en los servidores, las cuales procesan los datos mediante periciones, las cuales en su mayoria estan programadas en API REST, las cuales se encargan de conectarsen con las bases de datos, programando en gran medida en el lado del servidor

FRONTEND

Se relaciona con todo lo que ven las personas, esto incluye el diseño (bien sea UI o UX) y ciertos lenguajes. Además, destaca por su característica funcional, pues sirve para que el usuario entre e interactúe con cualquier sitio. Motivo por el que debe cumplir con altos estándares de usabilidad y estética. Podemos decir que el Front-end es la capa que se encuentra por encima del Back-end, ya que comprende menús desplegables, imágenes, íconos, colores, elementos gráficos, animaciones y, sin lugar a dudas, mejora la experiencia de navegación en el lado delantero o frontal. Quienes desarrollan el Front-end son responsables del código orientado a la inmersión del sujeto en el sitio.

SERVIDOR

En el software se ejecuta todo el backend, todo el desarrollo que tiene la lógica y las peticiones a las bases de datos, serán ejecutadas en el software del lado del servidor. Entre los servidores más populares y usados están NGINX, APACHE, NODE. Servidor toma la URL a la cual el usuario está solicitando la información o consulta, la cual, mediante un puerto definido por el desarrollador del servidor, será mostrada la información por lo general el puerto más utilizado para practicas es el puerto 80.

STACK

El significado de STACK es pila, son todos los paquetes de programación los cuales fueron desarrollados en su momento por el esquipo de desarrolladores para el backend, al ser un sistema de paquetes comprende la lógica de forma ordenada, por módulos y orientado a objetos. El STACK más popular es LAMP, el cual su significado es sistema operativo Linux, servidor Apache, base de datos MySQL y PHP, Existen diferentes tipos de stack, entre los cuales se encuentran MEAN Stacks, MERN Stacks, MEVN Stacks, LAMP Stacks. La funcionalidad consiste en enviar la URL al servidor, la cual mediante la DNS será transformada en una dirección IP, se conectará con el servidor en referencia a la URL, pasara por el backend, tomara la información de la base de datos y enviará todo al forntend para que el usuario pueda ver la información consultada.

Patrón MVC (modelo de vista del controlador)

En este patrón los datos se clasifican dependiendo de la funcionalidad, como es llamado este patrón, se separa en tres componentes lógicos los cuales interactúan entre si. Por lo general este patrón se utiliza cuando existen diferentes formas de interactuar con los datos. Permite que los datos mostrados cambien de forma independiente, en ocasiones es de gran complejidad cuando los modelos de datos y las interacciones entre ellas son simples.

CONTROLADOR

En los controladores tenemos la parte lógica del sistema que se encargara de ejecutar las acciones para la ejecución de la lógica del sistema, en los momentos en donde se requiera validar los datos se emplearan técnicas de eventos junto con JavaScipt y AJAX, así poder realizar una validación completa de los datos y manejarlos.

MODELO

Mediante la interaccion de las vistas y el usuario, esta le enviara al controlador las solicitudes para taer de los modelos la informacion de la base de datos, esto a su ves se conectan al servidor, mediante las diferentes peticiones traera la informacion solicitada

Patrón arquitectura en capas

En la arquitectura por capas, esta organiza los módulos como su nombre lo indica, por capas, las cuales se relacionan una con otra según su funcionalidad. A la hora de construirse nuevas funcionales, las capas se distribuyen en diferentes capas para cada equipo quien es el encargado de cada funcionalidad. Entre sus ventajas permite sustituir capas completas conservando sus interfaces.

Podemos tomar como ejemplo un sistema WEB, en el cual la capa superior corresponde a la interfaz gráfica de la página WEB de un sistema de libros, la segunda capa está compuesta por otras tres capas, una que se encarga de la conexión, el gestor formatos y consultas y el gestor de impresiones. En una tercera capa se podrían desarrollar los gestores de derechos, las recuperaciones de documentos, las búsquedas distribuidas entre otras. En la cuarta capa los índices de la biblioteca y por último en la quinta capa la base de datos.

Patrón arquitectura de repositorio

Para este caso cada cliente es independiente de los otros los cuales interactúan con el repositorio, el repositorio se encarga de dar respuesta a las peticiones de cada cliente, en cuanto a los usuarios estos serán los encargados de interactuar con los clientes. Como ejemplo tomaremos un banco, varios usuarios tendrán acceso al repositorio sin tener ningún problema, una vez el repositorio este bien definido se pueden agregar nuevos servicios en forma de subsistemas.

Entre las ventajas, sus componentes pueden ser independientes no necesitan conocer la existencia de otros componentes, los cambios realizados en un componente se pueden extender a otros componentes, todos los datos de pueden gestionar de una forma más fácil ya que su arquitectura se encuentra en un solo lugar. Entre sus desventajas, se encuentra que por tener un único repositorio las fallas que se presenten afectan todos los clientes, la distribución de sus componentes no diferentes equipos pueden ser compleja.

Patrón arquitectura de aplicación

Cada empresa cuenta con procesos similares, esto quiere decir, ejemplo si nos fijamos en la operación financiara de una empresa industrial, en cuanto a su contabilidad, la forma de llevar su contabilidad será similar solo cambiaran algunas especificaciones dependiendo del tipo de producto. Es decir, para toda empresa las forma de llevar sus costos es igual, tomar las materias primas, el personal, los insumos, los gastos, los honorarios y todo llevarlo a una base de información que establece el costo, la única diferencia es que en cada empresa los componentes y elementos que componen el conjunto de los costos en algunas será menor y con mayor facilidad de identificar y clasificar, en otras será más compleja y de mayor encapsulamiento para mantener la información real.

Tomaremos como ejemplo una aplicación WEB con arquitectura de aplicación con el framework Angular, mediante en la interfaz gráfica se presentaran los electos de diseño, los estilos y los eventos, los cuales estarán maquetados con CSS y HTML utilizando Web components, en cuanto al backend se desarrolla con los lenguajes de programación TypeScript de JavaScript para determinar los estados y ejecutar los eventos de frontend previamente maquetado con web components y por ultimo XMLHttpRequest el cual se encargara de los modelos, las conexiones a las bases de datos y puntos de red.
